Generative AI & Machine Translation Acknowledgement

This project involved strategic use of AI tools to accelerate development while maintaining critical evaluation of all generated content. This page documents which tools were used, how they were applied, and demonstrates critical engagement with AI outputs.

AI Tools Used

Claude Sonnet 4.0 & 4.5 (Anthropic)

Primary Use: Code generation, CSS architecture, accessibility guidance, and content structuring

Strengths: Strong understanding of semantic HTML, WCAG compliance, and modern CSS practices. Explaining technical concepts and providing context-aware code suggestions.

Limitations Identified: Initially suggested overly complex solutions that needed simplification. Sometimes included features beyond project scope (like localStorage that doesn't work in Claude.ai artifacts). Required strong and restrict constraints to match existing design system.

Critical Application of AI Tools

1. Code Debugging

What AI Did: Refine my previous code structure, fix CSS conflicting, some inappropriate image display, issues in image size, view and shadow effects. Fix and test JavaScript modules for API integration (GET/POST modules, form validation, dynamic content rendering).

Critical Evaluation Applied:

  • Rejected AI's first attempt at icon button CSS using complex filters that made icons invisible—debugged through iterative questioning and testing
  • Challenged AI when it suggested 24px icons were sufficient; insisted on 60px for accessibility based on user research showing elderly users need larger touch targets
  • Modified AI-generated form CSS to better match existing component skeleton system rather than using standalone styles
  • Identified when AI provided correct code but wrong explanation, or slight error in code when AI insists it is correct. (e.g., claiming form labels were missing when they were actually present)

2. Accessibility Implementation

What AI Did: Suggested WCAG compliance strategies, aria-label implementations, and heading hierarchy structures.

Critical Evaluation Applied:

  • Questioned AI when it claimed heading hierarchy was wrong—manually verified that h1→h2→h3 structure was semantically correct
  • Pushed back on AI's recommendation to change h3 to h2 in icon section, recognizing this would break proper hierarchy
  • Identified that accessibility checker flagged false positive; discuss whether I need to change strictly based on the checker. Ask AI to rewrite audit report to accurately reflect actual vs. perceived issues
  • Required AI to provide specific WCAG criterion numbers and explain user impact in terms of actual research participants (Akasha, Denisho, Fligo)

3. Content Writing & Documentation

What AI Did: Drafted values mapping explanations, design rationale annotations, and ethical analysis content.

Critical Evaluation Applied:

  • Corrected AI when it assumed community was "older adults only"—clarified project serves diverse ages (21-67) facing different isolation causes
  • Rejected generic design principles; demanded content grounded in actual user research quotes and specific pain points
  • Rewrote AI's academic tone to match professional client presentation style appropriate for CCI

4. Design System Development

What AI Did: Proposed CSS architecture, component patterns, and responsive images.

Critical Evaluation Applied:

  • Identified conflict between AI's suggestion to use flexbox and existing grid-based card layout—chose grid for consistency
  • Modified AI's image sizing from 100% width to fixed heights (220px) with object-fit to solve alignment issues AI didn't initially address
  • Requested multiple iterations on button states until visual distinction was clear enough for actual usability

Reflection on AI-Assisted Development

What Worked Well

AI tools significantly accelerated initial code generation and provided multiple implementation approaches to consider. Claude Sonnet 4.5's understanding of accessibility standards and semantic HTML was particularly valuable. The iterative dialogue format allowed progressive refinement—I could test outputs, identify issues, and request specific corrections rather than accepting first attempts.

AI's ability to explain technical concepts (like how aria-label works, why heading hierarchy matters) enhanced my understanding rather than just providing code to copy-paste. This educational aspect made me a more informed developer capable of evaluating future AI suggestions critically.

Limitations Encountered

AI tools lack visual understanding—they cannot "see" layout problems that are obvious in screenshots. Issues like misaligned images or inconsistent card heights required manual identification before AI could propose solutions. This gap meant I needed strong debugging skills; AI assistance was only useful after I correctly diagnosed the problem.

AI sometimes prioritized theoretical best practices over pragmatic constraints. It would suggest elaborate solutions when simpler approaches better fit the existing codebase. I had to repeatedly specify "follow the current system" rather than accepting AI's tendency toward over-engineering.

Context window limitations meant AI occasionally forgot earlier decisions or constraints. I maintained separate notes about project structure, color values, and design decisions to provide consistent context across multiple conversations.

Professional Implications

This experience demonstrated that AI tools are powerful accelerators but poor decision-makers. They excel at generating options and explaining concepts but cannot determine which solution best serves project goals, user needs, or ethical principles. That judgment remains human responsibility.

Critical evaluation skills became more important, not less, when using AI assistance. I needed to understand accessibility standards, CSS specificity, semantic HTML, and user research methodology well enough to recognize when AI outputs were incorrect, incomplete, or contextually inappropriate.

The most valuable skill wasn't prompt engineering—it was knowing when to reject AI suggestions entirely. Recognizing that "Network or server error" indicated VPN issues rather than code problems, or that accessibility checkers can flag false positives, required domain knowledge AI cannot provide.
